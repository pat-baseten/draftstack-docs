# Roles and Permissions

## Team Roles

### Team Admin
Team Admins have full control over project settings and configurations. The following actions require the Team Admin role (or Organization Admin role):

- Updating project settings
- Deleting projects
- Managing monitored repositories
- Configuring Linear integration
- Managing style guide categories
- Triggering documentation backfills
- Syncing style guide tokens

### Team Member
Team Members have read access to projects within the team but cannot modify project settings or configurations.

## Troubleshooting

### Missing Repositories
If you do not see your repository listed in the setup or configuration screens, you may need to update the GitHub App permissions or access configuration.

- **Update GitHub App permissions**: [Update GitHub App permissions](https://github.com/apps/draftstack/installations/new)
- **Configure GitHub App access**: [Configure GitHub App access](https://github.com/apps/draftstack/installations/new)
