# Custom Style Guide Categories

You can now create, edit, and delete custom style guide categories for your project. This allows you to define specific change types beyond the default set (e.g., General, API Change, New Feature) and tailor the style guide to your team's workflow.

## Managing Categories

To manage your style guide categories:

1. Navigate to the **Style Guide** page in your dashboard.
2. Click the **Manage Categories** button in the top right.
3. Use the **Add Category** button to create a new category.

### Creating a Category

When adding a new category, you will need to provide the following details:

- **Key**: A unique identifier for the category (e.g., `MY_CATEGORY`). The key must be uppercase and cannot contain spaces.
- **Label**: The display name for the category (e.g., "My Category").
- **Description** (Optional): A brief description explaining the purpose of this category.

### Editing and Deleting

- To edit a category's label or description, click the **Settings** icon next to the category in the list.
- To delete a category, click the **Trash** icon. Note that default categories cannot be deleted.

## Usage

Once created, your custom categories will appear in the change type selector dropdown. You can write specific style instructions for each category, just as you would with the default categories.

### Example

If you create a category with the key `SECURITY_AUDIT`, you can select it from the dropdown and define specific guidelines for security-related changes.
