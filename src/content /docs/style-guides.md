# Importing Style Guides

You can import style guides by uploading `.md` or `.mdx` files. This allows you to create or update style guide categories in bulk using drag-and-drop or file selection.

## Supported File Formats

The import feature supports the following file types:
- `.md`
- `.mdx`

## How to Import

1. Navigate to the **Style Guide** page within your project dashboard.
2. Drag and drop your files into the import zone, or click the zone to browse your file system.
3. The system will process the files and automatically create or update style guide categories.

## File Configuration

Style guide categories are generated based on the filename. You can optionally configure the category metadata using YAML frontmatter within the file.

### Category Key and Label

The category key and label are automatically derived from the filename:
- **Key**: Generated by converting the filename to uppercase and replacing spaces or hyphens with underscores (e.g., `my-style-guide.md` becomes `MY_STYLE_GUIDE`).
- **Label**: Generated by converting the filename to title case (e.g., `my-style-guide.md` becomes `My Style Guide`).

### Optional Frontmatter

You can override the default label, add a description, or set the display order by adding a YAML frontmatter block at the top of your file.

```yaml
---
label: "Custom Label"
description: "A detailed description of this style guide."
order: 1
---
```

**Supported Fields:**
- `label` (string): Overrides the auto-generated category label.
- `description` (string): Sets the category description.
- `order` (number): Determines the sort order of the category relative to others.
